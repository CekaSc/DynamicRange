# ALLEGRO EDM4HEP Analysis Pipeline

A Python-based analysis pipeline for FCCee ALLEGRO detector simulations in EDM4HEP format. The pipeline covers the full chain from particle gun simulation to final plots.

## Pipeline Overview

```
ddsim (simulation) → selection (EDM4HEP → TTree) → histograms → plots
```

Each step can be run independently or as a full chain.

## Requirements

- Python 3.10+
- ROOT (with PyROOT)
- podio
- dd4hep
- g4units
- PyYAML

```bash
pip install pyyaml
```

The remaining dependencies (ROOT, podio, dd4hep) are expected to be available via your FCC software environment, e.g.:

```bash
source /cvmfs/sw.hsf.org/key4hep/setup.sh
```

## Directory Structure

```
.
├── run.py                  # Main entry point
├── config.yaml             # All analysis parameters
├── plot_layers.cxx         # ROOT macro for plotting
├── analysis/
│   ├── __init__.py
│   ├── constants.py        # Physical and detector constants
│   ├── utils.py            # Shared calculation functions (ENC, drift velocity)
│   ├── simulation.py       # ddsim steering
│   ├── selection.py        # EDM4HEP -> flat TTree
│   ├── histogram.py        # TTree -> histograms
│   └── plotting.py         # Calls plot_layers.cxx
├── sim_output/             # ddsim output ROOT files (generated)
├── sel_output/             # Selection output ROOT files (generated)
└── plots/                  # Output figures (generated)
```

## Configuration

All parameters are controlled through `config.yaml`. Before running, edit the file to match your setup:

```yaml
simulation:
  particle: "e-"
  energy: "10*GeV"
  n_events: 100
  seed: 42
  compact_file: "$K4GEO/FCCee/ALLEGRO/compact/ALLEGRO_o1_v03/ALLEGRO_o1_v03.xml"
  output_dir: "sim_output"
  theta_bins:
    - [55, 65]
    - [65, 75]
    - [75, 85]
    - [85, 95]
    - [95, 105]
    - [105, 115]
    - [115, 125]

selection:
  output_dir: "sel_output"
  capacitance_file: "../capac/capacitances_perSource_ecalBarrelFCCee_theta_update2025.root"

histograms:
  hv_mode: 2        # 1: uniform HV, 2: split HV (layers 1-6 / 7-11), 3: fixed 10 kV/cm
  percentage: 100
  noise_file: "../capac/noise_capa_ecalbarrel/elecNoise_ecalBarrelFCCee_theta.root"

plots:
  output_dir: "plots"
  modes: [1, 4, 6, 7]   # see plot modes below
```

The `capacitance_file` and `noise_file` are generated by a separate capacitance computation package and must be provided by the user.

### Plot modes

| Mode | Content |
|------|---------|
| 1 | Energy deposit per cell [MeV] |
| 2 | Charge per cell [fC] |
| 3 | Current per cell [µA] |
| 4 | Sum energy deposit per layer [MeV] |
| 5 | Sum charge per layer [fC] |
| 6 | Sum SNR per layer |
| 7 | SNR per cell per layer |

### HV modes

| Mode | Description |
|------|-------------|
| 1 | Uniform HV across all layers |
| 2 | Split HV: 1.2 kV for layers 1–6, 1.8 kV for layers 7–11 |
| 3 | Fixed electric field of 10 kV/cm |

## Usage

### Full pipeline

```bash
python run.py --all
```

### Individual steps

```bash
python run.py --simulate    # Run ddsim for all theta bins
python run.py --select      # Run selection: EDM4HEP -> flat TTree
python run.py --histogram   # Compute histograms from TTree
python run.py --plot        # Produce plots from histograms
```

Steps can be combined freely:

```bash
python run.py --select --histogram
```

### Custom config file

```bash
python run.py --all --config my_config.yaml
```

## Output files

Selection and histogram results are written back into the same ROOT file per theta bin, named following the convention:

```
sel_output/<particle>_<energy>_<thetaMin>_<thetaMax>_output.root
```

For example:

```
sel_output/em_10*GeV_55_65_output.root
```

Each file contains the flat `Energy` TTree from the selection step, plus the histograms added by the histogram step.

Plots are saved as PDF files in `plots/`:

```
plots/em_10*GeV_55_65_Energy.pdf
plots/em_10*GeV_55_65_SumSNR.pdf
...
```

## Physical constants

All detector geometry, LAr properties, and conversion factors are defined in `analysis/constants.py`. Modify this file if running with a different detector configuration.

## Notes

- The `$K4GEO` environment variable must be set before running the simulation step.
- The `--simulate` step calls `ddsim` via subprocess and requires a working Key4hep environment.
- The `--plot` step calls `plot_layers.cxx` via ROOT's interpreter (`root -l -b -q`).
